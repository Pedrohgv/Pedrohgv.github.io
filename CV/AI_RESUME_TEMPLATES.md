# AI Resume Templates and Checklists

Use these templates with the skills installed in `.opencode/skills`.

## Monthly Update Template

```markdown
# Monthly Resume Update - YYYY-MM

## New Achievements
- [Project/initiative]
  - Action:
  - Impact:
  - Metric (exact or conservative estimate):
  - Evidence/source:

## Skills and Tech Stack Changes
- Added:
- Increased proficiency:
- Deprioritized/removed:

## Bullet Upgrade Candidates
- Current bullet:
- Improved bullet draft:
- Quantification added:

## ATS and Keyword Notes
- Target role families this month:
- Missing keywords to introduce naturally:
- Terms to reduce/remove:

## Approved Changes for CV/resume.tex
- [ ] Summary updated
- [ ] Skills section reordered
- [ ] Experience bullets updated
- [ ] Projects updated
- [ ] LaTeX compiles cleanly
```

## Monthly Update Checklist

- [ ] Run `resume-version-manager` to confirm baseline and variant strategy.
- [ ] Run `tech-resume-optimizer` for role-aligned structure and wording.
- [ ] Run `resume-bullet-writer` for top 5-10 bullets.
- [ ] Run `resume-quantifier` to add/verify metrics.
- [ ] Run `resume-ats-optimizer` for keyword and formatting checks.
- [ ] Apply approved edits in `CV/resume.tex`.

## Per-Job Checklist

- [ ] Run `job-description-analyzer` and capture match score.
- [ ] Run `resume-tailor` to adapt summary, skills, and bullet priorities.
- [ ] Run `resume-quantifier` for any missing measurable impact.
- [ ] Run `resume-ats-optimizer` to validate keyword coverage.
- [ ] Save variant details via `resume-version-manager` notes.
- [ ] Add or update one row in `CV/application-tracker.md`.
- [ ] Update `CV/resume.tex` only with truthful, validated claims.

## Notes on Lower-Priority Skills

- `cover-letter-generator`: use after resume variant is finalized.
- `linkedin-profile-optimizer`: run when profile drift appears.
- `interview-prep-generator`: use after interview invite.
- `salary-negotiation-prep` and `offer-comparison-analyzer`: use at offer stage.
